2.1 핵심 원리: 세상에서 가장 정교한 '다음 단어 맞히기' 게임 (15분)

"거대 언어 모델의 작동 원리는 놀라울 정도로 단순합니다. LLM의 핵심 임무는 주어진 문맥에서 가장 확률이 높은 다음 단어를 예측하는 것, 단 하나입니다."

🎯 다음 단어 예측 체험

다음 문장을 완성해보세요:

  • "하늘은 높고 말은 살찌는 천고..." → "마비!"
  • "백설공주를 도와준 일곱..." → "난쟁이!"
  • "내가 가장 좋아하는 색은..." → ?

LLM도 정확히 이와 같은 일을 하지만, 그 규모가 상상을 초월합니다!

3단계: 선택 및 반복

반복의 마법: LLM이 '파란색'을 선택하면, 이를 원래 문장에 덧붙여 새로운 문맥을 만들고, 다시 다음 단어를 예측하는 과정을 무한히 반복합니다.

1회차: "내가 가장 좋아하는 색은" → 파란색

2회차: "내가 가장 좋아하는 색은 파란색" → 입니다

3회차: "내가 가장 좋아하는 색은 파란색입니다" → .

🌟 창발적 속성 (Emergent Properties): 단순한 "다음 단어 예측"이 방대한 데이터와 거대한 모델 크기에서 복잡한 추론, 창작, 번역 능력으로 발현됩니다. 이는 인간이 직접 프로그래밍하지 않은 능력들이 자연스럽게 등장하는 놀라운 현상입니다.

2.2 Transformer 아키텍처의 핵심: Attention 메커니즘 (20분)

LLM의 핵심 기술인 Transformer 아키텍처와 Attention 메커니즘을 비전공자도 이해할 수 있도록 직관적으로 설명합니다.

🔍 Attention 메커니즘 이해

비유: 독서할 때 중요한 부분에 형광펜을 치는 것과 같습니다!

예시 문장 분석

"그 남자가 길을 건너다가 차에 치였다"

이 문장에서 "그"는 누구를 가리킬까요?

  • ✅ "남자"와 높은 연관성 (Attention 점수: 0.9)
  • ❌ "길"과 낮은 연관성 (Attention 점수: 0.1)
  • ❌ "차"와 낮은 연관성 (Attention 점수: 0.2)

Attention 시각화

색이 진할수록 높은 Attention 점수

🧠 Self-Attention의 3가지 핵심 요소

1. Query (질문) 🤔

"지금 처리하고 있는 단어"

예: "그"라는 단어가 현재 처리 중인 Query

2. Key (열쇠) 🔑

"문장 내 모든 단어들"

예: "그", "남자", "길", "차" 등 모든 단어들

3. Value (값) 💎

"각 단어가 가진 의미 정보"

예: "남자"의 의미 정보 (성별, 인간, 주어 등)

🎯 Attention 계산 과정:
1. Query와 모든 Key들 간의 유사도를 계산
2. 유사도 점수를 확률로 변환 (Softmax)
3. 각 Value에 확률을 곱해서 가중 평균 계산
4. 최종 결과가 해당 단어의 새로운 표현이 됨

🔄 Multi-Head Attention

여러 개의 Attention을 병렬로 실행하여 다양한 관점에서 단어 간 관계를 파악합니다.

  • Head 1: 문법적 관계 (주어-동사)
  • Head 2: 의미적 관계 (동의어-반의어)
  • Head 3: 위치적 관계 (앞-뒤)
  • Head 4: 감정적 관계 (긍정-부정)

🎭 Multi-Head 비유

한 편의 영화를 여러 관점에서 동시에 분석하는 것과 같습니다:

  • 🎬 연출 관점
  • 🎭 연기 관점
  • 🎵 음악 관점
  • 📸 촬영 관점

2.3 대화의 기술: 프롬프트 엔지니어링 심화 (20분)

"AI와 대화하는 것은 하나의 기술입니다. 내가 얻는 결과물의 품질은 내가 던지는 질문의 품질에 정비례합니다."

🏗️ 프롬프트 구성의 5가지 핵심 요소

여기에 생성된 프롬프트가 표시됩니다...

🚀 고급 프롬프트 기법

Chain-of-Thought (사고 연쇄)

기법: "단계별로 생각해보세요"

예시: "이 수학 문제를 단계별로 풀어보세요: 1) 먼저 주어진 조건을 정리하고, 2) 적용할 공식을 선택하고, 3) 계산 과정을 보여주세요."

Few-Shot Learning (예시 학습)

기법: 예시를 통한 학습 유도

예시: "다음과 같은 형식으로 요약해주세요:
입력: 긴 기사 내용
출력: • 핵심 포인트 1
• 핵심 포인트 2"

Role Playing (역할 연기)

기법: 특정 인물이나 전문가 역할 부여

예시: "당신은 20년 경력의 심리상담사입니다. 다음 상황에서 어떤 조언을 하시겠습니까?"

Temperature 조절

기법: 창의성 vs 일관성 조절

예시: "창의적인 아이디어가 필요하면 높은 온도(1.2), 정확한 정보가 필요하면 낮은 온도(0.3)"

2.4 LLM 생태계: 상업용 vs 오픈소스 모델 비교 (5분)

LLM이 개발되고 배포되는 두 가지 주요 방식을 비교하여 AI 업계 생태계를 이해합니다.

🏢

상업용 모델

특징

  • 장점: 즉시 사용 가능, 안정적 성능, 기술 지원
  • 단점: 사용료 발생, 제한된 커스터마이징
  • 대표 모델: GPT-4, Claude, Gemini
  • 적합한 사용자: 빠른 도입이 필요한 기업, 비기술 사용자
🌍

오픈소스 모델

특징

  • 장점: 무료 사용, 완전한 커스터마이징, 데이터 보안
  • 단점: 기술적 전문성 필요, 인프라 구축 비용
  • 대표 모델: Llama, Mistral, Qwen
  • 적합한 사용자: 연구자, 개발자, 특수 요구사항이 있는 기업

📊 선택 기준 매트릭스

기준 상업용 모델 오픈소스 모델
초기 비용 낮음 (API 사용료만) 높음 (인프라 구축)
운영 비용 사용량에 비례 고정 (서버 유지)
기술 역량 불필요 고급 기술 필요
데이터 보안 제한적 통제 완전한 통제
커스터마이징 프롬프트 수준 모델 수준
확장성 자동 확장 수동 관리
💡 선택 가이드:
• 스타트업/프로토타입: 상업용 API (빠른 검증)
• 대기업/보안 중시: 오픈소스 + 자체 인프라
• 교육/연구: 오픈소스 모델 (실험과 학습)
• 일반 업무: 상업용 서비스 (편의성 중시)
